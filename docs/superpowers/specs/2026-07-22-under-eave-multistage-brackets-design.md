# 德宁之殿檐下多级斗拱与梁枋设计

## 目标

依据用户提供的仰视实拍图，重构德宁之殿首层与二层檐下木构关系。首层和二层均只保留一排连续斗拱，但每个斗拱单元内部由多级斗、拱、昂叠涩组成；斗拱从下到上逐级加宽、加深并增加出跳。首层斗拱下增加连续承托横梁，二层在下层屋顶与上层斗拱之间增加转换横梁，并在斗拱顶部设置承檐横梁，使柱、梁、斗拱和屋面形成连续受力视觉。

## 参考图提炼

- 首层檐下是一排连续铺作，不是上下排列的多排独立斗拱。
- 每个铺作内部存在多级叠涩，底部尺度最小，越接近屋檐尺度越大。
- 首层斗拱下方有连续横梁，与柱头和阑额共同承托铺作。
- 下层屋顶上方存在二层转换梁枋，二层斗拱从梁枋起步并连接上层屋檐。
- 牌匾为竖向窄矩形，上下边框较厚、左右边框较窄，独立悬挂在二层正面斗拱前方。

## 斗拱单元结构

现有 `addBracketSet` 改为生成统一的四级铺作单元，首层和二层复用同一层级逻辑，仅通过整体尺寸与出跳系数区分：

1. 第一级为底部坐斗和短承托，横向尺度约为完整尺度的 `55%`。
2. 第二级增加短拱与第一道昂，横向尺度约为 `70%`。
3. 第三级增加长拱与第二道昂，横向尺度约为 `85%`。
4. 第四级为承檐拱与顶部托枋，横向尺度为 `100%`。

四级构件的高度、纵深和向檐口方向的出跳同步递增。每个构件记录 `userData.stage`，取值为 `1` 至 `4`；斗拱组记录 `userData.stageCount = 4`。补间铺作继续比柱头铺作略小，但四级相对比例保持一致。

## 首层檐下结构

- 首层正面、背面、左侧和右侧维持现有柱头铺作与补间铺作位置。
- 删除首层单元内部目前只有基础四构件的简化表现，统一替换为四级铺作。
- 在首层斗拱最低点下方增加四面连续承托横梁，横梁与柱顶、现有阑额相接。
- 斗拱最低构件落在承托横梁上，最高构件低于下层屋面，不得悬空或穿瓦。
- 首层整体仍只有一排斗拱，不增加第二排或第三排独立斗拱组。

## 二层檐下结构

- 删除现有 `lower`、`middle`、`upper` 三排独立斗拱矩阵，改成单排斗拱矩阵。
- 单排矩阵继续覆盖正面、背面、左侧和右侧，并保留柱头铺作与补间铺作。
- 在下层屋顶上方设置四面连续转换横梁，作为二层斗拱的起点。
- 单排四级斗拱从转换横梁向上逐级出跳，顶部通过四面连续承檐横梁与上层屋顶衔接。
- 二层仍不增加墙面或独立立柱；透过斗拱可看到完整的檐下层次。
- 二层斗拱组不再使用 `userData.tier` 表示独立排数，改用单元内部的 `userData.stage` 表示叠涩层级。

## 横梁命名与关系

为自动化验证和结构分层观察增加以下命名节点：

- `首层斗栱承托梁架`：首层四面连续承托横梁组。
- `二层斗栱转换梁架`：下层屋顶与二层斗拱之间的四面连续横梁组。
- `二层斗栱承檐梁架`：二层斗拱顶部与上层屋顶之间的四面连续横梁组。

三组梁架均作为 `grid` 的直接子组。斗拱仍放在 `brackets` 检查层，结构分层模式可以独立观察梁枋和斗拱的接触关系。

## 牌匾比例

- 牌匾底板改为竖向窄矩形，宽约 `2.8m`，高约 `3.8m`，高度与宽度比约为 `1.36`。
- 上下边框厚度约 `0.32m`，左右边框宽度约 `0.20m`，形成“上下厚、左右窄”的边框关系。
- 金色内边同步适配竖向底板，不恢复文字。
- 两侧垂花和顶部翼形装饰随新宽度收拢。
- 悬挂横梁宽度缩短，吊杆位置改到牌匾竖向中心线两侧，牌匾继续与斗拱保持正面间距。

## 自动化验收

- 首层斗拱数量与现有四面布置一致，每组 `stageCount` 为 `4`。
- 二层斗拱由现有 `144` 组降为单排 `48` 组，四个方向均包含柱头铺作与补间铺作。
- 二层斗拱不再存在 `lower`、`middle`、`upper` 三种独立 `tier`。
- 每个斗拱单元的四级横向包围盒满足 `stage 1 < stage 2 < stage 3 < stage 4`。
- `首层斗栱承托梁架`、`二层斗栱转换梁架` 和 `二层斗栱承檐梁架` 均存在且包含四面连续梁。
- 斗拱包围盒最低点与对应下横梁顶部相接，最高点低于对应屋面。
- 牌匾高度大于宽度，高宽比位于 `1.30` 至 `1.42` 之间；上下边框厚于左右边框。
- 二层围护墙和二层独立立柱保持不存在。

## 浏览器验收

- 正面：首层和二层均呈现一排内部多级铺作，横梁连续，牌匾为竖向窄矩形。
- 左侧：四面斗拱和三组梁架连续闭合，无构件悬空。
- 低机位仰视：可以清晰读出斗拱从下到上的四级出跳和尺度递增，下层屋顶与二层梁架之间没有断层。
- 旋转观察：斗拱不穿瓦，梁枋不穿柱，牌匾不嵌入斗拱。
- 浏览器控制台无 Three.js、WebGL、材质或几何错误。
