# 原生分辨率、柔化动态阴影与正面栏杆补段设计

## 目标

在不改变现有相机交互和建筑比例的前提下，完成三项局部调整：大殿主台基正面栏杆左右各向中心补一档；所有设备固定使用最高模型精度与原生设备像素比；阳光保持正面循环，并呈现更柔和但更容易观察到的移动阴影。

## 正面台基栏杆

- 保留现有左右两段主台基正面栏杆的位置和外侧端点。
- 在两段栏杆靠近中心的一端，各增加一个独立标准补段。
- 每个补段包含一块白石横板和一个新的中心侧端柱。
- 补段与原栏杆共用连接处的既有立柱，不生成重叠立柱。
- 补段长度使用当前正面栏杆的标准柱间距，使新增构件与原有节奏一致。
- 左右补段镜像布置，均沿主台基正面边缘向中心延伸。

## 固定最高画质

- `selectQuality` 对所有设备档案均返回 `high`，不再根据屏幕宽度、设备内存或像素比降档。
- 屋面瓦片、线条、斗拱和其他依赖画质档的几何体始终使用高精度参数。
- WebGL 渲染器始终启用抗锯齿和阴影。
- 渲染器像素比直接使用 `window.devicePixelRatio`，不设置 `2x` 上限；高像素手机按原生 `3x/4x` 像素比渲染。
- 阴影贴图固定使用高画质尺寸，不按设备降级。

## 柔化动态阴影

- 阴影贴图使用 Three.js 的柔化 PCF 模式，减轻生硬锯齿边缘。
- 太阳仍只在大殿正面的高亮角度范围内往返，不移动到侧后方。
- 适当降低直射光和半球补光强度，避免正面过曝，并增强柱、栏杆和斗拱投影的明暗可读性。
- 保持连续闭合的自然光循环；太阳左右位置持续变化，使投影在地面和建筑构件上平滑移动。
- 测绘光照模式保持不变。

## 测试与验收

- 几何测试确认左右各存在一个正面补段，每个补段恰好包含一块横板和一个新增端柱。
- 几何测试确认补段与既有栏杆无缝连接、左右镜像，并且只向中心延伸。
- 画质测试确认窄屏、低内存和高像素比设备仍选择 `high`。
- 渲染器测试或可测试配置确认原生设备像素比不被截断，柔化阴影配置始终启用。
- 光照循环测试确认太阳保持在正面范围、亮度降低、首尾闭合且左右位置持续变化。
- 浏览器分别使用桌面和手机视口检查构件无重叠、画布非空、页面无横向溢出和控制台错误。

## 风险边界

原生高像素比加高精度几何会显著增加部分手机的 GPU 与显存负担。本方案按已确认的 B 方案执行，不再自动降级；若设备 WebGL 能力不足，页面可能比原先更耗电或更慢，但不会在代码中回退到中低画质。
